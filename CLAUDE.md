# Karpathy 编程指南

## 1. 思考后再编码

**不要假设。不要隐藏困惑。呈现权衡。**

实施前：
- 明确陈述假设。若不确定，请询问。
- 如有多种解读，全部呈现——不要默认选择。
- 如有更简单的方法，请指出。必要时请反对。
- 如有不清楚之处，停下来。说出困惑之处并询问。

## 2. 简单性优先

**用最少代码解决问题。不投机取巧。**

- 不添加需求之外的功能。
- 不为单次使用的代码创建抽象。
- 不添加未请求的"灵活性"或"可配置性"。
- 不处理不可能场景的错误。
- 如果50行能解决却写了200行，请重写。

## 3. 精准修改

**只触碰必须修改的。清理自己的烂摊子。**

- 不要"改进"相邻代码、注释或格式。
- 不要重构未损坏的部分。
- 匹配现有风格。
- 每行修改的代码都应直接追溯到用户请求。

## 4. 目标驱动执行

**定义成功标准。循环验证直到通过。**

将任务转化为可验证的目标：
- "添加验证" → "为无效输入写测试，然后让测试通过"
- "修复bug" → "写能重现bug的测试，然后让测试通过"
